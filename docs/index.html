// 02 — ABOUT

The Problem with Cloud-Based Hacking Tools

Modern AI-assisted penetration testing platforms push your most sensitive data — network topologies, zero-day findings, vulnerability telemetry — to remote cloud APIs. For red teams, this is an unacceptable trade-off.

Kali MCP WebUI takes a different path. By orchestrating everything through a local Ollama-compatible LLM instance and the Model Context Protocol (MCP), the platform bridges natural language reasoning to native Kali Linux toolchains without a single packet crossing your perimeter.

The result: an autonomous agent that can run nmap, gobuster, nikto, Metasploit, and dozens of other tools — directed by conversational prompts, bounded by configurable network policies, and logged with forensic-grade transcripts — entirely offline.

// 05 — FINDINGS SNAPSHOT

Preliminary Results and Observations

In a multi-step pivoting scenario, the AgentFlow workflow successfully reached the target objective using Ollama with MCP-connected tools. Instrumented traces enabled phase-level analysis of time and interaction patterns across reconnaissance, service identification, exploitation, and pivot setup.

Pivot Test Setup (Figure 2)

  • Scenario: multi-step pivoting engagement with staged reconnaissance and internal reachability constraints.
  • Execution: Ollama-based agent with MCP-connected cybersecurity tools in a local environment.
  • Measurement: trace-derived phase timing and interaction counts across the attack workflow.

Successful End-to-End Execution

The agent completed the defined pivoting task while producing structured traces suitable for post-hoc analysis and replay.

Observed vs. Estimated Efficient Workflow

Trace-based analysis estimated roughly 50 minutes in the observed run versus about 11 minutes under a counterfactual efficient orchestration setting.

Where Time Concentrated

The largest overhead appeared in exploit selection, repeated exploit attempts, and pivot configuration phases.

Actionable Failure Patterns

Recurring patterns included redundant tool calls, command-context mismatches, exploit retry loops, and pivot setup complexity.

These observations are preliminary and drawn from controlled runs; they are intended to guide systematic workflow improvements rather than serve as broad statistical claims.

Conclusion and Next Steps

AgentFlow shifts evaluation from outcome-only benchmarking to execution-level observability. By embedding an LLM agent in a local, bounded tool environment and capturing full interaction traces, the framework enables more transparent diagnosis and iterative improvement of agent-assisted penetration testing workflows.

Future work expands scenario diversity and dataset scale, with increased automation and human-in-the-loop analysis to detect recurring inefficiencies and align agent behavior with established penetration testing practice.

// 07 — INSTALLATION

Up and Running in Minutes

01

Clone the Repository

git clone https://github.com/raistlinJ/kali-mcp-webui.git cd kali-mcp-webui
02

Install Python Dependencies

python3 -m venv venv && source venv/bin/activate pip install Flask mcp ollama
03

Start Ollama & Pull a Model

ollama serve ollama pull llama3

Any tool-calling model works. llama3.1, qwen2.5, and mistral-nemo are good choices.

04

Launch the Server

python3 app.py

Open http://localhost:5055 in your browser. Select a model, enable tools, and click Start Service.

05

Or Use Docker

docker-compose up --build

A Dockerfile and docker-compose.yml are included for containerized deployments.

Requirements

  • OS: Kali Linux (recommended) or any Debian-based distro with security tools installed
  • Python: 3.10+
  • LLM: Ollama or any Ollama-compatible endpoint
  • Model: Any tool-calling model (llama3, llama3.1, qwen2.5, mistral, etc.)
