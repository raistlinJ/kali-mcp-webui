// 01 — ABOUT

The Problem with Cloud-Based Hacking Tools

Modern AI-assisted penetration testing platforms push your most sensitive data — network topologies, zero-day findings, vulnerability telemetry — to remote cloud APIs. For red teams, this is an unacceptable trade-off.

Kali MCP WebUI takes a different path. By orchestrating everything through a local Ollama-compatible LLM instance and the Model Context Protocol (MCP), the platform bridges natural language reasoning to native Kali Linux toolchains without a single packet crossing your perimeter.

The result: an autonomous agent that can run nmap, gobuster, nikto, Metasploit, and dozens of other tools — directed by conversational prompts, bounded by configurable network policies, and logged with forensic-grade transcripts — entirely offline.

🔒
Air-Gap Ready

No internet required. No API keys sent to third parties. Optional local auth for internal proxies.

🤖
Agentic Reasoning

The LLM reads tool output and autonomously decides the next action — closing the loop between reconnaissance and exploitation.

📋
Audit-Grade Logging

Every tool call, every LLM turn, every annotation is timestamped and archived as structured JSON and Markdown transcripts.

// 02 — FEATURES

Built for Operators. Designed for Researchers.

Fully Localized Execution

Runs entirely on your machine or trusted VM. Credentials stay local and are never written to run logs. Supports Ollama, LiteLLM proxies, OpenAI-compatible endpoints, and Claude.

Agentic Tool Execution

The LLM autonomously triggers Kali utilities — nmap, gobuster, Metasploit, sqlmap, and more — via MCP and integrates raw output directly into its reasoning loop.

Synchronous Subprocess Blocking

Prevents LLM hallucinations by forcing the agent to wait for long-running processes to complete. The agent sees real output, not speculative completion.

Comprehensive Audit Trails

Generates structured JSON tool execution logs and human-readable Markdown transcripts for every session, downloadable as ZIP archives.

Human-in-the-Loop Annotations

Analysts insert timestamped notes mid-engagement to guide the agent or flag areas of interest. Annotations feed into live span analysis and post-mortem reviews.

Live Span Analysis

Analyze the last N minutes of a live engagement with one-shot LLM inference to generate rapid pivoting strategies or identify execution bottlenecks in real-time.

Post-Mortem Session Analysis

Feed completed session transcripts back into the LLM to auto-generate narrative success summaries, highlight critical vulnerabilities, and pinpoint methodological optimizations.

Network Policy Enforcement

Configurable allow/disallow lists gate every tool invocation by IP, CIDR, hostname, or URL — preventing scope creep at the MCP server layer, not just the UI.

Tool Watcher Agent

A background monitoring agent continuously watches live session logs and surfaces MCP tool opportunities and pivoting suggestions without interrupting the active engagement.

// 05 — INSTALLATION

Up and Running in Minutes

01

Clone the Repository

git clone https://github.com/raistlinJ/kali-mcp-webui.git cd kali-mcp-webui
02

Install Python Dependencies

python3 -m venv venv && source venv/bin/activate pip install Flask mcp ollama
03

Start Ollama & Pull a Model

ollama serve ollama pull llama3

Any tool-calling model works. llama3.1, qwen2.5, and mistral-nemo are good choices.

04

Launch the Server

python3 app.py

Open http://localhost:5055 in your browser. Select a model, enable tools, and click Start Service.

05

Or Use Docker

docker-compose up --build

A Dockerfile and docker-compose.yml are included for containerized deployments.

Requirements

  • OS: Kali Linux (recommended) or any Debian-based distro with security tools installed
  • Python: 3.10+
  • LLM: Ollama or any Ollama-compatible endpoint
  • Model: Any tool-calling model (llama3, llama3.1, qwen2.5, mistral, etc.)
